# 实验6 基于 OpenCV 的局部特征检测、描述与图像匹配
## 一、实验简介
本实验基于**ORB特征检测器**，通过调整核心参数 `nfeatures` 为 500、1000、2000，对比不同特征点上限对图像特征提取、特征匹配、错误点剔除及目标定位效果的影响；同时引入**SIFT特征算法**，完成 ORB 与 SIFT 两种经典局部特征的横向对比，分析两种算法在匹配精度、内点占比、定位鲁棒性、运行特性上的差异，掌握特征匹配、RANSAC 误匹配剔除、单应性矩阵求解与透视变换目标定位完整流程。

## 四、实验功能说明
### 1. ORB 多参数对照实验
1. **特征检测**：调用 `cv2.ORB_create()`，设置不同最大特征点数，检测并可视化绘制图像关键点，保存关键点效果图；
2. **特征粗匹配**：使用暴力匹配器 BFMatcher + 汉明距离匹配描述子，输出全部匹配结果与前50组优质匹配图像；
3. **RANSAC 误匹配剔除**：通过单应性矩阵结合 RANSAC 算法筛选正确匹配内点，计算**内点数量**与**内点比例**；
4. **目标定位**：利用透视变换映射模板图像角点，在场景图中绘制红色包围框，判断目标定位是否成功。

### 2. ORB 与 SIFT 对比实验
1. 基于 SIFT 算法完成特征提取、KNN 匹配 + Lowe 比率筛选优质匹配；
2. 同样使用 RANSAC 剔除误匹配、求解单应矩阵、实现目标检测定位；
3. 统计两种算法的匹配数量、内点比例、运行效率、定位效果，生成对比数据。

## 五、核心参数说明
1. `nfeatures`：ORB 算法最大提取特征点上限，数值越大，提取特征点数量越多，弱特征与噪声点占比越高；
2. **内点（inliers）**：RANSAC 筛选后符合透视变换模型的**正确匹配点**；
3. **内点比例**：$\text{inlier\_ratio} = \dfrac{\text{正确内点数量}}{\text{总匹配数量}}$，衡量匹配精准度；
4. 单应性矩阵：描述两张图像间透视变换关系，是目标跨场景定位的核心矩阵。

## 七、实验结果分析
### 1. 不同 nfeatures 对 ORB 实验的影响
- 随着 `nfeatures` 增大，模板图与场景图关键点数量、总匹配对数持续增多；
- 特征点上限越大，弱特征、重复纹理特征被大量提取，错误匹配大幅增加，**内点比例逐步下降**；
- 特征点并非越多越好：`nfeatures` 过高易受噪声干扰，单应矩阵求解偏差变大，会出现定位失败；适中参数（1000）可兼顾匹配数量与匹配精度，定位稳定性最优。

### 2. ORB 与 SIFT 算法对比
1. **ORB**：二进制描述子，计算速度快、实时性强，适合嵌入式与实时视觉任务，但高特征点数量下匹配鲁棒性一般；
2. **SIFT**：浮点型描述子，特征稳定性强、抗干扰能力强，内点比例更高，定位精度与鲁棒性更优，但计算量大、运行耗时更长；
3. 适用场景：实时轻量化场景选用 ORB，高精度、强鲁棒性需求场景选用 SIFT。

## 八、实验总结
本次实验完成了 ORB 特征参数对照实验与 ORB/SIFT 算法对比实验，完整实现了特征提取、特征匹配、误匹配剔除、单应矩阵求解、透视变换目标定位全流程。
实验验证了特征点数量需合理取舍，过多冗余特征会降低匹配质量；同时理解了二进制特征与传统浮点特征的优劣差异，为后续图像拼接、目标识别、视觉匹配等应用奠定基础。
